# Build a Database MCP Server with FastMCP and SQLite (and PostgreSQL)

This repository contains a production-quality implementation of a Database Model Context Protocol (MCP) server built with Python, FastMCP, and SQLite, with architectural support for PostgreSQL.

---

## 🌟 Features Implemented
1. **Server Foundation**:
   - Clean, modular code separated into database adapters (`implementation/db.py`), database seeding (`implementation/init_db.py`), and MCP server registry (`implementation/mcp_server.py`).
   - Unified `DatabaseAdapter` interface supporting both **SQLite** and **PostgreSQL**.
   - Automatic database initialization and seeding for SQLite if database is not present.
2. **Robust MCP Tools**:
   - `search`: Search records with columns, limit, offset, ordering, sorting direction, and complex parameterized filters.
   - `insert`: Insert records safely and return the newly created payload (including autogenerated IDs).
   - `aggregate`: Compute `COUNT`, `AVG`, `SUM`, `MIN`, `MAX` with optional group-by and filters.
3. **Dynamic MCP Resources**:
   - `schema://database`: Exposes the entire database schema mapping all user tables and columns.
   - `schema://table/{table_name}`: Dynamic URI template returning schema information for a single requested table.
4. **Safety and Input Validation**:
   - Whitelist validation for table names, column names, operators, and aggregate metrics.
   - Strict parameterized SQL query construction to prevent SQL Injection.
   - Rejects empty inserts.
5. **Bonus Capabilities**:
   - **SSE / HTTP Token Authentication**: Custom `TokenAuthMiddleware` that automatically enforces authentication under SSE transport when `MCP_SECRET_TOKEN` is set in the environment.
   - **Multi-Database Engine Support**: Full SQLite and PostgreSQL adapters behind a unified interface.
   - **Pagination Guidance & Capped Output**: Enforced limit constraints (maximum 100) to protect token output windows.
   - **100% Passed Test Suite**: Comprehensive unit tests and integration smoke tests included.

## 🚀 Setup Instructions

### 1. Install Dependencies
Ensure you have Python 3.10+ installed. Install the required libraries:
```bash
pip install fastmcp uvicorn sse-starlette
```
*(Optional) If you want to use PostgreSQL adapter, install `psycopg2` or `psycopg2-binary`:*
```bash
pip install psycopg2-binary
```

### 2. Initialize the Database
Run the DB initializer to create the database (`local.db`) with `students`, `courses`, and `enrollments` tables pre-loaded with seed data:
```bash
python implementation/init_db.py
```

### 3. Run Unit and Integration Tests
Verify the installation by running the test suite:
```bash
# Run 15 unit tests
python -m pytest implementation/tests/test_server.py

# Run the integration smoke test
python implementation/verify_server.py
```
*Expected output: `ALL SMOKE TESTS PASSED SUCCESSFULLY!`*

## 🛠️ Running the MCP Server

You can run the server in two modes:

### A. STDIO Transport (Local CLI Client, default)
```bash
python implementation/mcp_server.py
```

### B. SSE Transport (Network-based Remote Client)
```bash
python implementation/mcp_server.py --transport sse --host 127.0.0.1 --port 8000
```

#### 🔒 Enabling Token Authentication for SSE Mode:
Set the environment variable `MCP_SECRET_TOKEN` before starting the server:
```bash
# Windows PowerShell
$env:MCP_SECRET_TOKEN="my-secret-key"
python implementation/mcp_server.py --transport sse --port 8000
```
When this variable is set, the server rejects any incoming HTTP/SSE request unless it includes the header:
`Authorization: Bearer my-secret-key`.

## 🔍 Testing with MCP Inspector

MCP Inspector is an interactive web page to test your server's tools, schemas, and resource states.

Run the batch helper script to launch the inspector:
```bash
implementation/start_inspector.bat
```
This launches the MCP Inspector CLI running against the STDIO server. Open the printed localhost URL in your browser to interact with the GUI.

## 🔌 Client Configurations

Configure your favourite MCP clients using the examples below (replace `/ABSOLUTE/PATH/TO` with your actual workspace path):

### 1. Claude Code (`.mcp.json`)
Create a `.mcp.json` file in your home directory or current directory:
```json
{
  "mcpServers": {
    "sqlite-lab": {
      "type": "stdio",
      "command": "python",
      "args": ["/ABSOLUTE/PATH/TO/implementation/mcp_server.py"],
      "env": {}
    }
  }
}
```

### 2. Gemini CLI
Add the server alias using Gemini CLI:
```bash
gemini mcp add sqlite-lab /ABSOLUTE/PATH/TO/python /ABSOLUTE/PATH/TO/implementation/mcp_server.py --description "SQLite lab FastMCP server" --timeout 10000
```
Verify the server is connected and discoverable:
```bash
gemini mcp list
gemini --allowed-mcp-server-names sqlite-lab --yolo -p "Show the students in cohort B2 and their average grade."
```

### 3. Codex (`config.toml`)
Add to `~/.codex/config.toml`:
```toml
[mcp_servers.sqlite_lab]
command = "python"
args = ["/ABSOLUTE/PATH/TO/implementation/mcp_server.py"]
```
